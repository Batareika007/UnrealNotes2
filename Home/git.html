Это очень базовые команды для Git. Раздел не обновляется и служит только как архивная заметка.

GIT

Версия 5.4 GIT работает прекрасно, если не нужны фишки новых версий, то можно спокойно работать с гитом

версия 5.7 встроенный гит плагин сломан! постоянно будет показывать ошибку

объек is missing from this changelist (referenced from /..)

в двух словах в новых персиях очень сильно намудрили систему, она стала очень жестко проверять, на столько что теперь всегда что-то не хватает

можно убрать эту проверку, есть два пути:

открыть файл DefaultEngine.ini которая находися в папке Config

[ConsoleVariables] RevisionControl.FilesValidation.Enable=0

добавить эти строки если их нет

каждый раз в консоли писать

RevisionControl.FilesValidation.Enable 0

далее, если при нажатии кнопки view changes, выходит списко с тысячами объектов, можно отменить:

project settings - Editor - Revision Control - Enable Uncontrolled Changelist - убирать галочку

GIT — используется для подключения проекта к GitLab.

Source Control → Connect to Source Control

Provider → Git

В GitLab создаём простой репозиторий, копируем URL‑ссылку и вставляем её в Unreal Engine в поле URL. После подключения автоматически создаётся первый коммит.

Initialize project with Git — инициализирует проект под Git, ждём завершения.

Accept Settings — применяем настройки.

Indicator Source Control — зелёный значок означает, что проект подключён к системе контроля версий.

После каждого изменения в Content Browser появляется красная иконка — это значит, что объект изменён и готов к коммиту.

Чтобы сохранить изменения: Source Control → Submit to Source Control → пишем комментарий к коммиту.

После коммита можно отправить изменения на сервер командой Push.
