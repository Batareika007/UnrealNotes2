тогда к каждой можно будет подключить разную логику

выствить виджет на сцену

в акторе добавить компонент widget

Material

нужно создать специальный материал для виджета, чтоб он был виден на сцене

Spare - где будет виден виджет

world: как обычный актор, будет скрыт за объектами

screen: будет виден всегда на сцене, перекрывая другие объекты

widget class

класс виджета который мы выставляем на сцену

draw size

размер виджета

pivot

где будет находится от пивот поинт

Receive Hardware input

обязательно отменить чтоб можно было взаимодействовать мышкой

add child

можно добавить дочерний виджет в уже существующий на сцене

виджет с которым нужен интерактив так и называется widget interaction, его тоже добавляем на сцену, и там больше настроек для интеракции

widget ancer якор

будет стараться соблюдать соотношение сторон от якоря до объекта

clipping - чтоб объекты не вылазили за предел родителя, тем самым может обрезать контент, текст, картинки и тд

компоненты widget

  • background blur - сделать полупрозрачный полу blur
  • Menu Anchors - добвлять разного рода меню кнопки, подходит чтоб скрывать и показывать
    другие виджеты не надо добавлять в иерархию, только в деталях меню в отделе menu class
  • wrap box - добавляет боксы по порядку один за другим в определенном месте в выделеном заполнение
  • drag and dpor - виджет который можно поставить в любое место на экране(компонент виджета)
    создать не простой виджет а именно DragDropOperation
