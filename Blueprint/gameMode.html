GameMode — правила и логика игры

GameMode — это центральный класс, который определяет правила игры. Он существует только на сервере (или в одиночной игре) и управляет тем, как игра должна работать:

  • сколько у игрока жизней;
  • можно ли ставить игру на паузу;
  • ограничение по времени;
  • условия победы и поражения;
  • какой персонаж создаётся при старте;
  • какой контроллер используется;
  • какой HUD отображается;
  • какой GameState используется;
  • логика рестарта и респавна.

1. Основные настройки GameMode

В GameMode можно указать классы, которые будут использоваться в игре:

  • Default Pawn Class — какой персонаж создаётся при старте.
  • Player Controller Class — логика управления игроком.
  • HUD Class — интерфейс.
  • GameState Class — состояние игры (синхронизация в мультиплеере).
  • PlayerState Class — состояние конкретного игрока.
  • Spectator Class — камера наблюдателя.

Эти настройки определяют, как игра будет работать на фундаментальном уровне.

2. Tick Interval

Tick Interval — это интервал между тиками GameMode.

По умолчанию GameMode тикает каждый кадр, но можно задать интервал, например:

  • 0.1 сек — обновление логики 10 раз в секунду;
  • 1 сек — обновление раз в секунду (например, таймер раунда);
  • 0 — тик каждый кадр.

Это полезно для оптимизации, если логика не требует частого обновления.

3. Дополнительные параметры

В GameMode есть несколько важных флагов:

  • Use Seamless Travel — плавная смена уровней (важно для онлайна).
  • Start Players as Spectators — начать игру в режиме наблюдателя.
  • Pauseable — можно ли ставить игру на паузу.

Первые два параметра сверху — настройки для сетевой игры (онлайн).

4. Что GameMode НЕ делает

Важно понимать ограничения:

  • GameMode существует только на сервере (или в одиночной игре).
  • На клиентах GameMode отсутствует.
  • Для синхронизации данных используется GameState.

Поэтому любую информацию, которую нужно показывать игрокам, нужно хранить в GameState или PlayerState.

5. Где используется GameMode

  • логика начала и конца раунда;
  • респавн игроков;
  • ограничение по времени;
  • подсчёт очков;
  • условия победы;
  • режимы игры (Deathmatch, Survival, Arena);
  • управление жизненным циклом уровня.

GameMode — это «директор игры», который определяет, как игра должна работать.
