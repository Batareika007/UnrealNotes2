UX / UI — создание интерфейса в Unreal Engine

UX/UI — это система интерфейсов, меню, кнопок, окон и элементов, с которыми взаимодействует игрок. В Unreal Engine интерфейс создаётся через Widget Blueprint.

1. Создание UI Widget

Создаём новый User Widget (например, BP_UIWidget).

Внутри виджета есть два режима:

  • Designer — визуальное создание интерфейса.
  • Graph — логика кнопок, событий и функций.

значек а графе показывает что это только для интерфейса

2. Canvas Panel — основа интерфейса

Canvas Panel — это главный контейнер, в котором можно размещать любые элементы UI. Он позволяет свободно позиционировать объекты.

В Canvas можно добавлять:

  • кнопки;
  • текст;
  • картинки;
  • панели;
  • контейнеры (Box, Grid, Overlay).

3. Anchor Points — якоря

Anchors определяют, как элемент будет вести себя при изменении разрешения экрана.

Примеры:

  • центр экрана;
  • левый верхний угол;
  • правый нижний угол;
  • растяжение по всей ширине/высоте.

Правильные Anchors — ключ к адаптивному UI.

4. Контейнеры UI

Контейнеры автоматически размещают элементы внутри себя.

Vertical Box / Horizontal Box

Элементы выстраиваются вертикально или горизонтально.

Grid Panel

Позволяет размещать элементы по строкам и столбцам.

5. Показ и скрытие UI

Чтобы показать UI в игре, нужно создать виджет и добавить его в Viewport.

Чтобы управлять UI, создаём переменную-ссылку на виджет.

Теперь можно:

  • показывать UI;
  • скрывать UI;
  • менять текст, картинки, кнопки.

6. Input Modes — управление вводом

Input Mode определяет, кто получает управление: игра или UI.

  • Game Only — управление персонажем, мышь скрыта.
  • Game and UI — UI работает, но персонаж не двигается.
  • UI Only — только UI, игра не реагирует.

8. Логика кнопок

В Designer выбираем кнопку → добавляем событие OnClicked.

В Graph появляется событие, где можно написать логику.

Пример: увеличить HP персонажа.

9. Настройки кнопок

В кнопке можно настроить:

  • анимацию наведения;
  • звук клика;
  • цвета состояний;
  • иконки;
  • стили.

Если звук не работает — проверь настройки аудио.

11. Лучшие практики UX/UI

  • Используй Anchors для адаптивного интерфейса.
  • Храни UI в GameInstance, если он должен переживать смену уровней.
  • Показывай UI через PlayerController.
  • Используй Input Mode для меню и паузы.
  • Не перегружай Canvas — используй контейнеры.
  • Делай UI анимированным (Opacity, Scale, Move).
  • Добавляй звуки и визуальные эффекты кнопкам.

UX/UI — это лицо игры. Чем лучше интерфейс, тем комфортнее игроку.
