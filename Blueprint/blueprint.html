Blueprint

ConstructionScript – логика выполняется один раз в игре как только актор появляется на сцене. В редкаторе логика будет срабатывать каждый раз как только мы выставляем актор на сцену, и как то его редактируем

Event Graph – основаня логика актора

Event Graph – для каждого blueprint есть свои начальные функции, например

Event begin play – что будет происходить с самого начала жизни актора

Event tick – что будет происходить каждый фрейм, каждый раз когда рендерит новый кадр, если 60 фреймов, то в секунду произведёт 60 раз.

Event actorBeginOverlap – что происходит при прикосновение к blueprint - когда заходят за коллизию

** отключить соединения, держать alt на клавиатуре

** коментарии - выделить ноды и нажать на “С”

Variables – переменные – сведенья о blueprint – создаем новую переменную «+»

Variable types:

Boolean – true / false

Byte – числено от 0 - 255 , занимает мало памяти

Integer – целое число без остатка, положительно и отрицательное от -2билиона до +2 билиона, использует много памяти

Integer64 – очень большое число с 16 нулями, очень много памяти

Float – дробное значение, занимает больше памяти чем integer

Name – кусок текста обозначающее имя чего либо, по размеры самый легкий, мало памяти, не может быть изменен, переведен, нельзя манипулировать

String – кусок текста который можно изменять размер, шрифт и тд. Занимает больше памяти

Text – кусок текста который может быть переведен на любой язык, локализация, занимает очень много памяти, если тебе нужна локализация то ты пользуешь текст.

Vector – 3Д структура которая хранит x, y, z типа float

Rotator – три типа float для поворота по осям x, y, z

Transform – имеет размер и поворот и вектор по осям x, y, z

Functions – программа, которая включает в себя логику

особые функции для blueprint, например если у нас в одном blueprint два кубика, для каждого можно дать собственный эффект задачу,

Выбираем объект и его действия, пкм ‘collapse to function’ – создаем новую функцию которую можно вызвать в любое время

Spawn actors - Призвать блюпринт на карте вместо того чтоб выводить из content browser

В blueprint карты выбираем новый нод spawn actor from class

И в классе выбираем наш blueprint который хотим призвать

Array - массив в блюпринтах очень полезен, при создании ноды, пишем Array и он покажет нам все возможности что можно сделать с массивом, вот некоторые из них:

создаём массив и мы можем получи из него данные, найти данный, узнать длину массива, найти последний элемент, проверить если массив содержит данные, добавить в массив, удалить из массива по индексу или содержанию, очистить весь массив, соединить два массива.

Flow control:

Branch - условие которые выводит в true или false

Быстрое создание, держим ‘B’ и нажимаем ПКМ

Sequence - будет проигрывать один за другим всё что мы подключим, всё производится мгновенно без задержек, но по очереди

For Loop - производит постоянные действия несколько раз, зависит от индекса который мы задаём. Loop body - то что будет отражаться, Completed - то что произойдёт в конце нашего лупа.

While loop - тоже самое что и for loop, но он происходит до тих пор пока правдивое состояние Так же есть что будет происходить каждую итерацию и что будет по окончанию. Нельзя ставить состояние которое будет крутиться бесконечно, получим ошибку бесконечного лупа

Do N - ограничить выполнения действий N раз, то есть нажать на кнопку 3 раза и после этого действие не будет срабатывать. В примере, выполнится три раза каждый тик, и больше не будет выполнять, при нажатие на F снова выполнится три раза

Do Once - сделает только один раз за всё время запуска кода Start closed - не будет сразу запущенно если это позволяет

Flip flop - при каждом вызове будет менять из одного параметра на другой Is A - будет вызывать значение булион если это вариант A-true или false 

  
        Gate - как открытые ворота. Мы можем выбрать если они изначально закрыты или открыты.
        Enter - запускает ноду, если ворота открыты то пойдет дальше, если закрыты, то дальше не пойдет
        Open - открывает ворота, теперь enter - будет работать
        Close - закрывает ворота - Enter не работает 
        Toggle - закрывает и открывает  ворота
    


        Multi Gate - похожи на гейт только больше параметров 
        При вызове этой ноды, каждый раз будет производится действие по порядку которые мы назначим
        Значени loop и random - будет заново повторять все действие и рандомально запускать, но рандомально запускать каждый раз всю последовательность и только потом начинать заново.
        Мы не сможем рандомально вызвать первый и опять первый, а только когда он пройдет все данные, тогда рандомально начнёт заново.
        Reset -
        обнуляет gate и начинает с нуля.

Delay - запустит действие спустя назначеное время

Retriggerable Delay - запустить действие спустя назначенное время, но если запустить действие опять, то время обнулится и начнется с начала
