Event Dispatchers

Event Dispatcher — это механизм, который позволяет одному Blueprint уведомлять другие Blueprint о том, что произошло какое‑то событие. Это похоже на систему подписок: один объект «кричит», а другие «слушают» и реагируют.

В отличие от Cast и Interfaces, Event Dispatcher создаёт гибкую связь: объект, который вызывает событие, не знает, кто именно его слушает. Это делает систему более чистой и независимой.

Как работает Event Dispatcher

  1. Один Blueprint создаёт Dispatcher — это «сигнал».
  2. Другие Blueprint подписываются на этот сигнал (Bind).
  3. Когда событие происходит, вызывается Call — и все подписчики получают уведомление.

Это идеальный способ сообщать о событиях:

  • когда объект должен уведомить несколько других объектов;
  • когда слушатели могут появляться и исчезать;
  • когда нужно избежать жёстких связей между Blueprint;
  • когда один объект должен запускать логику в других, но не знает их классов.

Call — вызвать событие

Это команда, которая запускает Dispatcher. Blueprint, который «кричит», вызывает Call — и все подписчики получают сигнал.

Пример: дверь сообщает всем слушателям, что она открылась.

Bind — подписать слушателя

Blueprint, который хочет реагировать на событие, подписывается на Dispatcher. После Bind он будет получать уведомления каждый раз, когда вызывается Call.

Пример: лампочка подписывается на событие «дверь открылась».

Unbind — отключить одного слушателя

Удаляет подписку конкретного Blueprint. После этого он перестаёт реагировать на Dispatcher.

Пример: лампочка перестаёт слушать дверь.

Unbind All — отключить всех слушателей

Blueprint, который вызывает Dispatcher, очищает весь список подписчиков. После этого никто больше не будет получать уведомления.

Пример: дверь перестаёт рассылать уведомления всем объектам.

Event — обработчик события

Это событие создаётся в Blueprint, который слушает Dispatcher. Когда вызывается Call, этот Event запускается автоматически.

Пример: лампочка получает сигнал и включается.

Assign — быстрый способ подписаться

Assign — это комбинация Bind + Event. Он одновременно создаёт обработчик события и подписывает Blueprint на Dispatcher.

Это удобный способ быстро подключить слушателя, не создавая Event вручную.
