Прямая коммуникация

Прямая коммуникация — это самый простой способ одного Blueprint обратиться к другому. Мы создаём переменную-ссылку (Object Reference) на нужный Blueprint и вызываем его функции или меняем переменные напрямую.

Такой способ удобен, когда:

  • объекты заранее известны (например, персонаж и дверь);
  • оба Blueprint находятся на уровне;
  • нужна простая и быстрая связь без интерфейсов и кастов.

1. Создаём Blueprint, с которым хотим взаимодействовать

Например, это может быть дверь. Никаких специальных настроек не требуется — обычный Blueprint Actor.

2. В Blueprint персонажа создаём переменную-ссылку

В персонаже создаём новую переменную и в типе выбираем Blueprint двери (Object Reference). Это позволит нам обращаться к его функциям и переменным.

  • Тип переменной должен быть именно Door_BP (Object Reference).
  • Переменная должна быть Instance Editable — включаем «глазик».

3. Добавляем логику в Event Graph

Теперь можно вызывать функции двери напрямую. Например, открыть дверь при нажатии кнопки.

В Event Graph персонажа вызываем переменную двери → вызываем нужную функцию (например, OpenDoor).

4. Настраиваем ссылку на уровне

Чтобы переменная работала, нужно вручную указать, какой именно объект на уровне является нашей дверью.

Для этого:

  1. Выбираем персонажа на уровне.
  2. В Details находим нашу переменную.
  3. Выбираем объект двери из списка.

Теперь персонаж знает, с какой дверью он должен взаимодействовать.

Когда прямая коммуникация НЕ подходит

Прямая ссылка удобна, но имеет ограничения:

  • Если объект создаётся динамически (Spawn Actor), ссылка может быть неизвестна заранее.
  • Если объектов много (например, 20 дверей), вручную назначать ссылки неудобно.
  • Если Blueprint должен работать с разными типами объектов — лучше использовать Interface.
  • Если связь должна быть двусторонней — Event Dispatchers работают лучше.

Поэтому прямая коммуникация — это хороший вариант для простых, статичных ситуаций.
