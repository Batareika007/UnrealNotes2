Localization — система локализации текста в Unreal Engine

Локализация — это процесс перевода всех текстов проекта на разные языки. Unreal Engine использует мощную систему, которая автоматически собирает все строки, хранящиеся в формате Text, и позволяет переводить их через Localization Dashboard.

Важно: локализация работает только с типом данных Text. Строки типа String не участвуют в переводе.

1. Открываем Localization Dashboard

Dashboard — это центр управления локализацией. Здесь создаются языки, собираются тексты и выполняется компиляция переводов.

2. Настройка источников текста (Gather Text)

Нужно указать, где движок будет искать тексты для перевода.

Обычно выбирают папку Content или конкретные директории, где находятся UI, диалоги, квесты и т.д.

3. Сбор текста (Gather Text)

После настройки нажимаем Gather Text.

Процесс может занять время — движок ищет все Text-поля в проекте.

Внизу появится отчёт:

Здесь видно, сколько строк найдено и сколько нужно перевести.

4. Добавление языков (Cultures)

Нажимаем Add New Culture и выбираем язык, например:

  • en — English
  • ru — Russian
  • fr — French
  • de — German

После добавления снова нажимаем Gather Text, чтобы обновить список строк.

5. Перевод строк

В списке появятся две колонки:

  • Source (оригинальный язык)
  • Target (новый язык)

Выбираем язык → нажимаем Edit Translations.

В открывшемся окне переводим каждую строку вручную.

6. Компиляция переводов

После завершения перевода нажимаем Compile Translations.

Это создаёт файлы локализации, которые будут использоваться в игре.

Кнопка Count Words показывает статистику перевода.

7. Проверка перевода в виджетах

В любом Widget Blueprint сверху есть выбор языка. Можно переключать язык и проверять, как UI реагирует.

Важно: иногда изменения не отображаются сразу — нужно перезапустить редактор.

8. Смена языка в игре (Set Current Culture)

Чтобы переключить язык во время игры, используем ноду:

Set Current Culture

Вводим код языка:

  • en
  • ru
  • fr

После смены языка UI обновится автоматически.

Важно: в редакторе язык UI не меняется — только в Standalone или в собранной игре.

String Tables

String Tables — это таблицы строк, которые позволяют хранить текст по ключам. Это лучший способ локализовать большие проекты.

Принцип работы:

  • каждая строка имеет ключ;
  • в коде или UI используется ключ;
  • локализация переводит строки автоматически.

1. Создание String Table

Или через поиск → String Table.

В таблице:

  • Key — уникальное имя строки;
  • String — текст, который будет отображаться.

2. Использование String Table в UI

В любом Text поле можно выбрать:

  • String Table
  • Key

Теперь текст будет автоматически переводиться через Localization Dashboard.

3. Рекомендации по String Tables

  • Используйте таблицы для диалогов, квестов, описаний предметов.
  • Создавайте отдельные таблицы для разных систем (UI, предметы, квесты).
  • Не используйте String — только Text.
  • Не меняйте ключи после перевода — это ломает локализацию.

4. Финальный процесс локализации

  1. Создать String Tables (если нужно).
  2. Открыть Localization Dashboard.
  3. Настроить источники текста.
  4. Gather Text.
  5. Add Cultures.
  6. Перевести строки.
  7. Compile Translations.
  8. Проверить в игре через Set Current Culture.

После этого проект полностью готов к многоязычной поддержке.
