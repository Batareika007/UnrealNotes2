Camera shake тряска камеры

Представь, что камера висит на пружинках. Эти настройки определяют, насколько эти пружины жесткие и в какие стороны они тянут камеру.

root shake pattern - выбираем wave oscillator camera shake pattern

у каждого параметра есть Initial Offset: Откуда начинается тряска при запуске.

Zero: Всегда из центра.

Random: С любого места (чтобы тряска не выглядела одинаково каждый раз).

location

Здесь ты настраиваешь, как камера двигается вперед-назад (X), влево-вправо (Y) и вверх-вниз (Z).

Amplitude (Амплитуда): На какое расстояние (в сантиметрах) камера может отойти от центра.

Для ходьбы: ставь по Z (вверх-вниз) небольшое значение, например 2.0.

Frequency (Частота): Как быстро камера колеблется.

Для Idle (дыхание): ставь 0.2.

Для бега: ставь 10.0 и выше.

Rotation (Поворот)

Самый важный пункт для "ощущения" веса. Это повороты камеры вокруг осей.

Pitch: Наклон вверх-вниз (как кивок головой). Хорошо для имитации шага.

Yaw: Поворот влево-вправо. Можно добавить чуть-чуть для ползания.

Roll: Наклон к плечам. Самый важный параметр для ходьбы. Когда мы шагаем, наше тело слегка наклоняется влево-вправо. Если поставить Roll на 0.5, картинка сразу станет "живой".

Timing

Duration (Длительность):

Если 0, то тряска будет длиться вечно, пока ты её не остановишь кодом (идеально для ходьбы и дыхания).

Если > 0, она проиграется один раз (для удара или прыжка).

Blend In / Blend Out: Время "разгона" и "торможения" тряски.

Если поставить 0.2, то при начале ходьбы камера не дернется мгновенно, а плавно начнет раскачиваться. Это критически важно для мягкости!

2. Рекомендуемые значения для состояний

Состояние Amplitude (Сила) Frequency (Частота) Совет
Idle (Дыхание) 0.1 - 0.5 0.2 - 0.5 Используй только Pitch и Roll
Шаг (Walk) 1.5 - 3.0 6.0 - 8.0 Основной упор на Location Z и Roll
Бег (Run) 4.0 - 6.0 10.0 - 14.0 Добавь немного Perlin Noise
Ползание (Crawl) 2.0 - 4.0 3.0 - 4.0 Медленно, но с большой амплитудой в Yaw
