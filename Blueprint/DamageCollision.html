Damage Collision

Damage Collision — это система, которая позволяет объектам реагировать на столкновения и получать подробную информацию о том, что именно произошло: кто ударил, куда, с какой силой и каким компонентом. Это основа для урона, эффектов, физики и интерактивных объектов.

1. Simple и Complex Collision

Simple Collision — простые примитивы (Box, Sphere, Capsule). Они дешёвые, быстрые и подходят для физики и блокировок.

Complex Collision — точная коллизия, полностью повторяющая форму меша. Она дорогая по производительности и используется только для трассировок (LineTrace).

2. Collision Volume — объём коллизии

Collision Volume — это невидимая форма, которая определяет границы объекта.

  • Показывает, где находятся границы актора.
  • Определяет форму для столкновений.
  • Нужен для физики — без него объект не сможет участвовать в симуляции.

В каждом Static Mesh можно просмотреть и изменить коллизии, а также переключаться между Simple и Complex.

3. Управление коллизиями в Static Mesh Editor

Удаление и добавление коллизий

Remove Collision — удалить все коллизии.

Add Box / Sphere / Capsule — добавить простые примитивные коллизии.

10DOP / 18DOP / 26DOP — автоматические оболочки разной детализации. DOP создаёт форму, сглаженную по выбранным осям X/Y/Z.

Auto Convex Collision — создаёт сложную коллизию, максимально повторяющую форму меша. Чем выше параметры, тем точнее и тяжелее коллизия.

4. Damage Collision — включение событий столкновения

Чтобы объект мог сообщать о столкновениях, на его компоненте нужно включить:

Simulate Generate Hit Events

После этого в Event Graph появится событие Event Hit, которое вызывается каждый раз, когда объект сталкивается с чем‑то.

5. Параметры Event Hit

Event Hit возвращает подробную информацию о столкновении:

  • My Comp — компонент нашего актора, который получил удар.
  • Other — другой актор, который столкнулся с нами.
  • Other Comp — компонент другого актора, участвующий в столкновении.
  • Self Moved — показывает, кто двигался.
    true — наш объект двигался и ударил другой.
    false — другой объект ударил нас.
  • Hit Location — точные координаты X/Y/Z точки удара.
  • Hit Normal — нормаль поверхности в точке удара (с какой стороны пришёл удар).
  • Normal Impulse — сила столкновения.
  • Hit — структура с расширенной информацией о столкновении.

6. Где используется Damage Collision

  • Система урона (Damage System).
  • Отталкивание, физические реакции.
  • Эффекты попаданий (искры, кровь, следы).
  • Определение стороны удара (для анимаций и логики).
  • Разрушаемые объекты.
  • Хитбоксы оружия.

Event Hit — это один из самых мощных инструментов в Blueprint, потому что он даёт полный набор данных о столкновении, позволяя создавать реалистичную и гибкую игровую логику.

Damage Collision - урок от колизий

На любой меш ставим галочку simulaty generate hit event Далее в event grapgh у нас будет event hit

  • My comp - какой компонент этого актера был задет.
  • Other - другой меш который столкнулся с нашим актером
  • Other comp - вернет компонент другово меша
  • Self moved - булеан который будет показыват если сам объект бьёт что-то, если наш персонаж тот кто бьет этот обект, то будет false
  • Hit location - точные данные X Y Z где произошёл удар
  • Hit normal - вернёт векторные данные, с какой стороны произошёл удар
  • Normal Impulse - насколько сильно было столкновение
  • Hit - дает ещё больше информации о столкновение
