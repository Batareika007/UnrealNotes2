Float track - стандартный. Есть ось X - это время (сколько будет длится действие) и ость Y это значение которые мы получаем каждый фрейм.

Vector Track - создает три вектора xyz их по одному можно настраивать по отдельности, так же есть глазик и замочек, что было легче редактировать, закрывать или скрывать

На выходе получае вариант с векторами

Так же можно его разбить на три оси

Event Track - у value нет значения, интересуют только время, в котором будут происходить события, например каждую секунду персонаж прыгал на протяжение 3 секунд. Грубо говорят, Update делает каждый фрейм, а Event каждый промежуток который мы задаём

Color Track - назначает и меняет цвета во времени, например можно назначит градиент и он будет меняться на назначенном времени, в течени трех секунд будет происходить переменна цвета

Timer - для примера сделаем восстановление здоровья.

В основном пользуются двумя таймера- timer by event, timer by function name

Timer by Event - таймер будет вызван опеределлым евентом

Создаем эвент востановления здоровья, тригер на сцене на который будет наступать.

Далее создаем ещё два евента, при входи на тригер и выход с тригера, и создаем таймер, при входе запускаем таймер который будет повторятся и работать каждую 0.5 секунд и останавливаем таймер при выходе, в остановке мы подключаем какой именно таймер надо остановить.

Так же останавливаем таймер в бранче, как только здоровья дошла до максимума, иначе будет постоянно показывать что здоровье полное без остановки.

Чтоб всё это работало надо указать в нашем тригере каст на персонажа

Timer by Function - для этого надо создать функцию, напишем туже логику только уже внутри функции. Данный вариант немого почище и удобней для чтения

И в самом BP назначаем создаем таймер и даем ему имя функции, так как в предыдущем один раз это для входа на тригер и для выхода

В саму функцию тоже добавляем clear timer by function
