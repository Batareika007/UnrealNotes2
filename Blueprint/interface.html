Interfaces

Blueprint Interface — это способ общения между Blueprint без прямых ссылок, Cast и жёстких зависимостей. Интерфейс позволяет вызвать одну и ту же функцию у разных объектов, даже если они не связаны между собой и имеют разные классы.

так же это набор одной или более функций, только именныйх без назначения

Интерфейс работает как:

  • Event, если функция не имеет выходных значений (Outputs);
  • Function, если есть Outputs — тогда он возвращает результат.

1. Создание интерфейса

В Content Browser → Blueprint → Blueprint Interface. Внутри интерфейса создаются только функции — без логики, только сигнатуры.

Даём функции имя — именно по этому имени она будет вызываться в других Blueprint.

2. Подключение интерфейса к Blueprint

Чтобы Blueprint мог реагировать на вызов интерфейса, нужно подключить его:

  1. Открываем Blueprint.
  2. Переходим в Class Settings.
  3. В разделе Interfaces нажимаем Add и выбираем наш интерфейс.

После подключения в Event Graph появится возможность добавить Event интерфейса.

Добавляем Event и прописываем логику.

Или открываем вкладку интерфейса двойным кликом.

3. Взаимодействие через интерфейс

Теперь любой Blueprint может вызвать интерфейсную функцию у объекта, если у него есть ссылка на этот объект (например, через LineTrace, Overlap, SphereTrace).

Пример: персонаж делает Sphere Trace → находит объект → вызывает у него функцию интерфейса.

4. Inputs — входные параметры

В интерфейс можно добавить входные параметры (Inputs). Например: здоровье, мана, ID предмета, сила удара и т.д.

Эти параметры будут видны во всех Blueprint, которые используют интерфейс.

5. Outputs — превращение интерфейса в функцию

Если добавить выходные значения (Outputs), интерфейс перестаёт работать как Event и начинает работать как Function — то есть возвращает результат.

После этого во всех Blueprint интерфейсная нода изменится — теперь это Function Call.

Старый Event нужно удалить и заменить новой функцией.

Внутри функции можно настроить любую логику. Например:

  • если мана < 200 → добавить 50;
  • если мана ≥ 200 → ничего не делать.

А в вызывающем Blueprint можно получить возвращаемое значение.
