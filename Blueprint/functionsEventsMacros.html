Functions, Events, Macros

Events

Event — это реакция на какое‑то событие в игре: игрок блокирует атаку, подбирает предмет, нажимает кнопку и т.д. Можно запускать много событий одновременно, и одно событие может вызывать разные действия. Events не возвращают результат, они просто выполняют логику.

* В Blueprint ноды Events обычно с красным заголовком.

Custom Event

Можно создать свой собственный Event: задать ему имя и повесить на него нужную механику. После этого этот Event можно вызывать в других местах Blueprint, например — при нажатии клавиши.

Functions

Function — это кусок логики, который выполняется мгновенно и возвращает результат, которым можно дальше пользоваться. Функции помогают не дублировать один и тот же код много раз и сильно экономят время.

Public — функцию могут вызывать другие Blueprint.

Protected — функцию можно вызывать только в этом Blueprint и его наследниках.

Private — функцию можно вызывать только внутри текущего Blueprint.

Pure — функция без Exec‑пинов, не меняет состояние (переменные), только считает и возвращает значение.

Создать функцию можно через вкладку Functions, нажав на +, задать ей название и написать внутри логику.

* Важно: Pure‑функции не должны изменять переменные, только возвращать результат.

После создания функцию можно вызвать в Event Graph как обычную ноду. Визуально она похожа на Event, но у неё нет Exec‑пинов (если это Pure).

Также можно создать функцию из уже готового кода: выделить нужные ноды, ПКМ → Collapse to Function.

Есть два типа функций: Pure и Impure.

Pure — нельзя подключить Exec‑провода, используется только для вычислений и выдачи результата.

Можно создать библиотеку функций: Blueprint → Blueprint Function Library. Все функции из такой библиотеки будут доступны во всех Blueprint‑ах проекта.

Macros

Macro — это небольшая логика, которую нужно многократно повторять внутри одного Blueprint. Макросы удобны для визуальной чистоты и небольших часто используемых паттернов.

Collapse Nodes — можно взять большой фрагмент логики и свернуть его в один узел, чтобы упростить визуальное восприятие.

После Collapse Nodes получается одна компактная нода, которая внутри содержит всю прежнюю логику.
