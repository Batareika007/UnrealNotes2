Режимы коллизии

No Collision — нет коллизий. Объект словно призрак: существует на сцене, но не имеет физической формы, не участвует в трассировках, не вызывает Overlap и никак не взаимодействует с игроком (нет Probe).
Применение: декорации, эффекты дыма/огня — объект полностью игнорируется.

Physics Only — объект участвует только в физике. В него можно врезаться, на него можно встать, если у него есть гравитация — он будет катиться, его можно толкать. Но он не участвует в трассировках и не вызывает Overlap (нет Probe).
Применение: физические ящики, камни, которые можно толкать — участвуют в физике, но не в логике взаимодействия.

Query Only — объект не имеет физической формы, но участвует в логике: реагирует на Overlap, Line Trace, Sphere Trace и т.д. Физики нет, но можно создавать взаимодействия (нет Probe).
Применение: триггер‑зоны, хитбоксы оружия, зоны взаимодействия — участвуют в трассировках и Overlap, но не в физике.

Collision Enabled (Query + Physics) — универсальная коллизия. Есть и физика, и взаимодействие, и данные столкновений (есть Probe).
Применение: персонажи, враги, интерактивные объекты — полный набор: физика, блокировка, Overlap, трассировки.

Probe Only — объект получает данные о столкновении (где именно произошло касание), но не блокирует и не участвует в физике. Со всем остальным ведёт себя как No Collision.
Применение: IK ног, вспомогательные капсулы для анимации — нужны только контактные данные, без физики и блокировки.

Probe and Query — объект получает данные о точке столкновения и участвует в логике (Overlap, Line Trace), но не участвует в физике. Это как продвинутый Query Only: если Query может только определить факт пересечения, то Probe+Query позволяет узнать точное место контакта и реагировать на него.
Применение: хитбоксы персонажей, попадания пуль с эффектами — трассировки + контактные данные.

Object Type — кто я?

Object Type (Object Channels) определяет роль объекта в системе коллизий.

  • WorldStatic — неподвижные элементы окружения (стены, пол).
  • WorldDynamic — движущиеся объекты по скрипту (двери, лифты).
  • PhysicsBody — физические тела (ящики, камни).
  • Pawn — персонажи и NPC.
  • Vehicle — транспорт.
  • Destructible — разрушаемые объекты. которые меняют форму (PhysicsBody) так не может

Collision Responses — как я реагирую на других

  • Block — объект блокирует другой.
  • Overlap — объект не блокирует, но сообщает о пересечении.
  • Ignore — объект полностью игнорирует другой.

Collision Presets — готовые настройки

  • BlockAll — блокирует всё.
  • OverlapAll — пересекается со всем.
  • Pawn — стандарт для персонажей.
  • Trigger — для триггер‑зон.
  • PhysicsActor — для физических объектов.
  • Custom — свои комбинации Object Type + Responses.

Advanced — дополнительные настройки

  • Simple vs Complex Collision — примитивы против точной геометрии.
  • Physics Materials — материалы поверхности (шаги, звуки, эффекты).
  • Collision Filtering — настройка реакций объекта на каждый канал.
  • CCD (Continuous Collision Detection) — защита от «проскальзывания» быстрых объектов.
  • Sub-Stepping — более точная физика при низком FPS.
  • Async vs Sync Traces — оптимизация трассировок.

Simple и Complex Collision

Simple and complex collision – простая примитивная коллизия, и сложная повторяющая полностью объект
Можно задать, чтобы движок всегда использовал только простую коллизию:
Project Settings → Engine → Physics → Simulation → Default Shape Complexity: Use Simple As Complex.

Тогда все объекты будут использовать простую коллизию, и при выборе Default в Static Mesh Editor будет применяться именно она.

Управление коллизией в Static Mesh Editor: можно настроить, изменить или добавить новую.

Типы коллизий в Static Mesh Editor

  • Первые три, добавить простые коллизии сферу, капсулу, квадрат
  • 10DOP — создаёт бокс и сглаживает его по выбранным осям X/Y/Z.
  • 18DOP / 26DOP — более детализированные оболочки.
  • Convert boxes to convex - разделяет все квадраты на треугольники - преващает примитивные в сложные, чтоб обработка колизий была единой, если уже в существует сложная колизия и мы добавляем новую, её тоже надо сделать сложной.
  • Convert Boxes to Convex — превращает примитивы в сложные оболочки (треугольники). Если у объекта уже есть сложная коллизия, новые тоже должны быть сложными.

Convex Decomposition:

  • HULL count - на сколько частей разбить коллизию, будет несколько отдельных коллизий
  • Max hull verts - сколько вершин будет в каждой
  • Hull precision - на сколько детально будет коллизия

Double Sided Geometry

Работает только с Complex Collision.
когда у нас один объек прозрачный, скозь него не проёдет луч. А если мы поставим галочку то будет проходить тоже.

Never Needs Cooked Collision Data

не сохранять данные в готовом билде игры, это освобождает память, делаеть билд более лёгким, но это надо делать если ты уверено что данные объект будет только как дикорация и не более, никакой физики и не трасировок.

Customized Collision

Она появляется, когда у меша есть своя уникальная коллизия, созданная вручную или импортированная из DCC‑программы (Blender, Maya, 3ds Max).

Complex Collision Mesh

Можно взять другой меш и использовать его сложную коллизию, например сделать более легкую версию одного и того же, чтоб облегчить нагрузку, для оптимизации.

Complex Collision → даёт ту же точность, но дороже по производительности и не работает для физики.

небольшой итог

Simple Collision

  • Что это: примитивы (Box, Sphere, Capsule, Convex Hull), которые ты задаёшь вручную или генерируешь в редакторе.
  • Физика (Physics Bodies, Rigid Bodies, движение персонажей).
  • Столкновения объектов.
  • Оптимизация — очень быстрые в расчётах.
  • Минус: форма грубая, не всегда точно повторяет меш.

Complex Collision

  • Что это: сам меш используется как коллизия (Use Complex as Simple).
  • Трассировки (Line Trace, Raycast, хитсканы оружия).
  • Физика — не работает для симуляции физических тел (движок не использует полигоны меша для физики).
  • Плюс: идеально точное совпадение с геометрией.
  • Минус: очень тяжёлое для производительности, нельзя использовать для динамики.
