Collison

No collision - нет коллизий, объект словно призрак, существует на сцене, но ничем не обработан, никак не взаимодействует с игроком (нету Probe) -- Декорации, эффекты дыма/огня -- объект полностью игнорируется

Physucs Only - только физический объект, можно в него врезаться, встать, если объект имеет гравитацию, то катится, его можно толкать, но никак с ним не взаимодействовать, это не интерактивный объект. (нету Probe) -- Физические ящики, камни, которые можно толкать -- участвует в физике, но не в трассировках.

Query Only - объект, с которым можно взаимодействовать, но без физической формы, реагирует на overlap и line trace, можно создавать логику (нету Probe) -- Триггер‑зоны, хитбоксы оружия, зоны взаимодействия -- участвует в трассировках/оверлапах, но не в физике.

Colision Enabled - универсальна коллизия, есть всё и физика, и взаимосвязь и данные столкновения (есть Probe) -- Персонажи, враги, интерактивные объекты -- полный пакет: и физика, и трассировки.

Probe Only - получает данные о столкновении, где именно было столкновения, но мы не можем с ним взаимодействовать, со всем остальным он работает как No collision. -- IK ног, вспомогательные капсулы для анимации -- только контактные данные (без блокировки и физики).

Probe and Query - получает данные, где именно было столкновения и с ним можно взаимодействовать, он как продвинутый Query Only, если Query может узнать перешли коллизию или нет, то здесь мы может узнать, где именно это произошло и сделать выводы и совершить какие-либо действия -- Хитбоксы персонажей, попадания пуль с эффектами -- трассировки + контактные данные, без физики.

Object Type - (Object Channels / роли) кто я

  • WorldStatic → неподвижные элементы окружения (стены, пол).
  • WorldDynamic → движущиеся объекты по скрипту (двери, лифты).
  • PhysicsBody → физические тела (ящики, камни).
  • Pawn → персонажи и NPC.
  • Vehicle (виикел) → транспорт.
  • Destructible → разрушаемые объекты.

Collision Responses - как другие объекты реагируют на меня

  • Block → объект блокирует другой.
  • Overlap → объект не блокирует, но сообщает о пересечении.
  • Ignore → объект полностью игнорирует другой.

Collision Presets

  • BlockAll → блокирует всё.
  • OverlapAll → пересекается со всем.
  • Pawn → стандарт для персонажей.
  • Trigger → для триггер‑зон.
  • PhysicsActor → для физических объектов.
  • Custom Presets → твои комбинации Object Type + Responses.

Advanced (дополнительно)

  • Simple vs Complex Collision → примитивы vs точная геометрия.
  • Physics Materials → материалы поверхности (шаги, звуки, эффекты).
  • Collision Filtering → настройка реакций объекта на каждый канал.
  • CCD (Continuous Collision Detection) → защита от «проскальзывания» быстрых объектов.
  • Sub-Stepping → точная физика при низком FPS.
  • Async vs Sync Traces → оптимизация трассировок.

Simple and complex collision – простая примитивная коллизия, и сложная повторяющая полностью объект

Можно задать чтоб движок постоянно использовал только простую коллизию

Project settings – Engine – Physics – simulation - Default Shape Complexity: use simple as complex

Все объекты будут использовать простую, и когда в объектах будешь выбирать default это то что будет использовано

Управление в самом объекте, настроить колизию, изменить, добавить новую.

Collision: примитивные и сложные

  • Первые три, добавить простые коллизии сферу, капсулу, квадрат
  • 10DOP - создаёт квадрат и смягчает его по осям которым мы выбираем X Y Z
  • 18\26 DOP - более детализированные коллизии
  • Convert boxes to convex - разделяет все квадраты на треугольники - преващает примитивные в сложные, чтоб обработка колизий была единой, если уже в существует сложная колизия и мы добавляем новую, её тоже надо сделать сложной.

Convex Decomposition:

  • HULL count - на сколько частей разбить коллизию, будет несколько отдельных коллизий
  • Max hull verts - сколько вершин будет в каждой
  • Hull precision - на сколько детально будет коллизия

Duble sided geometry - работает только с complex collision - когда у нас один объек прозрачный, скозь него не проёдет луч. А если мы поставим галочку то будет проходить тоже.

Never Needs Cooked Collision Data - не сохранять данные в готовом билде игры, это освобождает память, делаеть билд более лёгким, но это надо делать если ты уверено что данные объект будет только как дикорация и не более, никакой физики и не трасировок.

Customized Collision

Она появляется, когда у меша есть своя уникальная коллизия, созданная вручную или импортированная из DCC‑программы (Blender, Maya, 3ds Max).

Complex Collision Mesh

Можно взять другой меш и использовать его сложную коллизию, например сделать более легкую версию одного и того же, чтоб облегчить нагрузку, для оптимизации.

Complex Collision → даёт ту же точность, но дороже по производительности и не работает для физики.

небольшой итог

Simple Collision

  • Что это: примитивы (Box, Sphere, Capsule, Convex Hull), которые ты задаёшь вручную или генерируешь в редакторе.
  • Физика (Physics Bodies, Rigid Bodies, движение персонажей).
  • Столкновения объектов.
  • Оптимизация — очень быстрые в расчётах.
  • Минус: форма грубая, не всегда точно повторяет меш.

Complex Collision

  • Что это: сам меш используется как коллизия (Use Complex as Simple).
  • Трассировки (Line Trace, Raycast, хитсканы оружия).
  • Физика — не работает для симуляции физических тел (движок не использует полигоны меша для физики).
  • Плюс: идеально точное совпадение с геометрией.
  • Минус: очень тяжёлое для производительности, нельзя использовать для динамики.
