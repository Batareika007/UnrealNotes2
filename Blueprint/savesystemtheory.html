Game Save System

чтоб сохранить прогресс, нужно создать новый класс save game и в нём создавать переменные с которыми будем работать, это могут быть любые перменные любых типов

движок не может сохранить всё как есть, мы сохраянем отдельные акторы, переменные в них, если дверь открыта или сколько здоровья осталось у игрока, а так же настройки графики. и при каждой загрузки игры, мы должны читать файл сохранения и применять к объекта в игре.

простыми словами, каждый объект на который должен влиять сохранения, в начале игры, проверяет, существует ли сохранения, получает к этому сохранению доступ, и применяет логику которую мы прописываем.

разберем ноды:

Does Save Game Exist

Does Save Game Exist это физическая проверка есть ли .sav файл на диски. физическое название этого файла это и есть - slot name

мы физически проверяем существует ли сохранение, и взависимости от ответа, ДАЛЕЕ создаем файл или получаем к нему доступ, но на этом этапи ничего не создается и не читается, только проверка.

Create Save Game Object

Create Save Game Object - Тут мы выбираем класс, который создали для сохранения. Это создание «чистого листа» в оперативной памяти...

Это создание «чистого листа» в оперативной памяти. Файл на диске ещё НЕ создан. Движок просто выделяет кусочек памяти и говорит: «Вот временный объект, он выглядит как твой актор savegame. Можешь записывать в него данные, пока игра запущена». Если ты выключишь игру на этом этапе, ничего не сохранится.

Load Game from Slot

Load Game from Slot - назначаем имя слота - это физическое название с файла .sav то что физическу существует в папке. и теперь она доступна для чтения. следующий этап этот файл мы может прочитать, взять с него информацию или записать.

Cast to savegame

cast to savegame - важно, чтоб прочитать этот файл (записать в него информацию или получить) делаем cast в актор savegame который создали в самом начале. в этом акторе находятся переменные которые нам нужны. без каста мы не увидим что есть в этом файле

на этом этапе мы или записываем новые данные в ОБЪЕКТ в памяти или читаем их, и изменяем логику нашего актора

Save Game to Slot

Save Game to Slot сама важная нода для сохранения, до этого момента все изменения которые вводи в файл, будет существовать пока не выключаем игру. в неё подлюкчает то что выходит из каста.

примечание - эту ноду вызываем только если что-то изменили и хотим зафиксировать это на диске. если читаем, то она не нужна.
