Trigger

Триггер — это невидимая зона, которая реагирует на вход, выход или нахождение объектов внутри неё. Чаще всего используется для запуска событий: открытие дверей, включение света, кат-сцены, диалоги, появление врагов и т.д.

1. Установка триггера на сцену

Перетащите Box Trigger, Sphere Trigger или Capsule Trigger на сцену. После этого обязательно выделите триггер в Outliner.

Это важно, потому что Blueprint уровня показывает события только для выбранного триггера. Если триггер не выбран — нужные ноды просто не появятся.

2. Настройка событий триггера в Level Blueprint

Откройте Level Blueprint. Теперь вы можете добавить события триггера:

  • OnActorBeginOverlap — вызывается, когда актор входит в триггер.
  • OnActorEndOverlap — вызывается, когда актор выходит из триггера.

Эти события позволяют запускать любую логику: включать анимации, менять переменные, запускать таймеры, проигрывать звук и т.д.

3. Пример использования

В этом примере:

  • Когда игрок входит в триггер — запускается одно действие.
  • Когда выходит — запускается другое.

Это базовый шаблон для дверей, зон загрузки, подсказок, появления врагов и любых других событий, которые должны происходить при входе/выходе из области.

4. Важные нюансы

  • Триггер должен иметь Collision Preset: OverlapAll или Trigger.
  • Игрок должен быть настроен так, чтобы вызывать Overlap (обычно Pawn → Overlap).
  • Если событие не срабатывает — проверьте коллизии.

Триггеры — один из самых простых и мощных инструментов в Blueprint, позволяющий создавать интерактивность без сложной логики.
