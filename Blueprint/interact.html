Интерактивность с объектами - делаем по середине экрана точку для интерактивности

Создаём blueprint interface

Называем, к примеру interact interface и в самом блюпринте создаём функцию interact

Далее создаём input кнопку на которое будет происходить действие

Так же нам надо 2–3 картинки указателя. Кружок, крестик...

Далее создаем Widget - назовем Pointer и туда вставляем картинку делаем по центру

Далее в нашем персонаже создаем блюприн настройки на event tick

Создаём переменную типа актор

Модифицируем NODE

Подключаем к INPUT нашу логику

В event begin play - подключаем виджет

И подключаем наш созданный widget, делаем promote to variable. Так же называем нашу новую переменную например pointer widget

И если всё сделано правильно, при старте уже должен быть виден наш pointer

Далее прописываем логику - если мы сталкиваемся с чем может интерактировать, то будет один pointer, например кружок с точкой внутри, а всё остальное время, то есть, когда мы не можем ни с чем интерактировать, будет пустой кружок

Следующий этап — это добавлять объекты, с которыми можно манипулировать Например, сделаем дверь, которая будет открываться от нажатия

Создаем пустой blueprint, добавляем static mesh и заменяем SceenRoot

Далее создаем переменную Boolean - open door

Далее заходим в Class Settings - там добавляем наш созданные interact interface

У нас появляется новые interface

Что нам дает возможность создать новый Event Interact

Создаем логику - если дверь закрыта то открыть, если открыта то закрыть

Лучше из timeline добавить lerp vector - что указать расстояние не трогая весь timeline

Далее чтоб для каждого актора на сцене не писать ту же логику, мы просто будет его заменять нашим главным актером, подставляя дочерний элемент

Создаем ещё одну переменную, типа static mesh object

В construction script - создаем логику замеы наш меш на новый и обязательно новые меш должен быть публичным (открытый глазик)

На этом этапе мы закончили.

Теперь берем уже существующию дверь или создаем новую и там заменяем меш на дочерний, который с логикой.

В новом blueprint добавляем дочерний актор - childActor

В details выбираем child actor class - актор с логикой который сделали

И в default выбираем меш нашей двери.

И в конце удаляем оригинальную дверь если они имеласть в начале... То есть мы заменили существующую дверь на интерактивный актор, и там добавили новую дверь.

Всё теперт всё должно работать, такими де дейсивиями добавляем логику отддельно для правой двери, для левой, выдвежных ящиков и тд...

Interface - использовать переменные для изменения чего либо

  1. В самом Enteract Event добавляем переменные, и каждый раз, когда вызываем event, он будет пользоваться этой переменной
  2. Вместо добавления переменной, добавляем ссылку на персонажа character – object reference. Это не действенный персонаж, а любой character. В input так и назовем character ref
  3. Создаем новый интерфейс, он будет этим самым character ref. Функцию называем так какое действие должно выполниться, например добавлять очки. В output - добавляем переменную уже с параметром, который нужно добавить
  4. В нашем персонаже добавили переменную, которая должна увеличивать счет. В персонаже подключаем настраиваем новый интерфейс - открываем его и у нас открывается функция
  5. В актор с которым мы коммуницируем из character ref - выводим нашу функцию и там уже есть переменная, которая будет изменена

Итог - вместо того чтоб наш интерфейс имел кучу переменных, мы вызываем только те которые нам нужныы
