Level Instance и Packed Level Actor — повторно используемые части уровня

Level Instance и Packed Level Actor позволяют взять группу акторов на уровне и превратить их в отдельный, повторно используемый элемент. Это удобно для модульного левел-дизайна, оптимизации и аккуратной структуры проекта.

Примеры использования:

  • повторяющиеся здания, комнаты, коридоры;
  • декоративные блоки (рынок, двор, лагерь, база);
  • модульные куски города или интерьера;
  • повторяющиеся сцены с одинаковым наполнением.

1. Что такое Level Instance

Level Instance — это отдельный уровень (Sub-Level), который вставлен внутрь другого уровня как объект. Он хранит внутри себя акторы, которые можно редактировать в отдельном Level Asset.

Ключевые особенности:

  • содержит полный набор акторов (мэш, свет, декор, триггеры и т.д.);
  • редактируется как отдельный уровень;
  • может использоваться многократно на разных картах;
  • поддерживает Level Streaming и World Partition.

Это лучший вариант, если тебе нужно модульно собирать уровни из повторяющихся блоков.

Это целый уровень, то есть там есть всё, и маши и блюпринт, и свет, это целый большой уровень

2. Что такое Packed Level Actor

Packed Level Actor — это «упакованная» версия Level Instance, где несколько акторов объединяются в один оптимизированный актор.

Ключевые особенности:

  • объединяет несколько статических объектов в один;
  • уменьшает количество draw calls;
  • улучшает производительность;
  • менее удобен для редактирования, чем обычный Level Instance.

Это больше про оптимизацию, чем про удобство редактирования.

3. Создание Level Instance из акторов

Шаги:

  1. Выдели на уровне группу акторов, которые хочешь превратить в модуль.
  2. ПКМ по выделению → Create Level Instance (или похожий пункт меню).
  3. Выбери путь и имя нового Level Asset (например, LI_Market).
  4. На уровне останется один объект — Level Instance, который ссылается на этот Level Asset.

Теперь этот Level Instance можно копировать, перемещать, вращать и использовать на других уровнях.

4. Редактирование Level Instance

Чтобы изменить содержимое Level Instance:

  1. Выбери Level Instance на уровне.
  2. Нажми Edit или Open (в Details или через ПКМ).
  3. Откроется отдельный уровень с его содержимым.
  4. Вноси изменения (добавляй/удаляй акторы, меняй свет, декор и т.д.).
  5. Сохрани уровень.

Все экземпляры этого Level Instance на других картах обновятся автоматически.

5. Создание Packed Level Actor

Packed Level Actor обычно создаётся на основе Level Instance или группы статических акторов.

  1. Выдели группу акторов или Level Instance.
  2. ПКМ → Create Packed Level Actor (название может отличаться в разных версиях UE).
  3. Движок создаст новый актор, который содержит «упакованную» версию геометрии.

Важно: Packed Level Actor — это больше про оптимизацию, чем про редактирование. Если нужно часто менять содержимое — лучше использовать обычный Level Instance.

6. Когда использовать Level Instance, а когда Packed Level Actor

Level Instance — когда:

  • нужны повторяющиеся модули уровня;
  • ты хочешь редактировать содержимое отдельно;
  • один и тот же блок используется на разных картах;
  • нужна поддержка Level Streaming / World Partition.

Packed Level Actor — когда:

  • сцена тяжёлая и нужно уменьшить количество акторов;
  • нужно оптимизировать рендер (draw calls);
  • содержимое редко меняется;
  • это финальный декор, а не геймплейная логика.

7. Практический пример: модульная комната

  1. Собираешь комнату из стен, пола, мебели, света.
  2. Выделяешь всё → Create Level Instance.
  3. Сохраняешь как LI_Room_A.
  4. Копируешь Level Instance по уровню, поворачиваешь, комбинируешь.
  5. Когда всё готово и нужно оптимизировать — создаёшь Packed Level Actor (опционально).

В результате у тебя:

  • чистая структура уровня;
  • повторно используемые модули;
  • меньше шансов что-то сломать;
  • возможность быстро менять дизайн одной комнаты во всех местах.

8. Лучшие практики

  • Давай понятные имена Level Instance (LI_…): LI_Market, LI_Room_A, LI_Tower_01.
  • Храни их в отдельной папке, например Content/Levels/Modules.
  • Не смешивай в одном Level Instance геймплей и чистый декор, если это можно разделить.
  • Используй Level Instance для модульности, Packed Level Actor — для финальной оптимизации.
  • Перед упаковкой в Packed Level Actor убедись, что дизайн уже финальный.

Level Instance и Packed Level Actor — это мощные инструменты модульного левел-дизайна и оптимизации. Они позволяют строить большие миры аккуратно, повторно и эффективно.
