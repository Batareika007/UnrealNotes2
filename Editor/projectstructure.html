пример организации проекта готов

Структура проекта

Структура проекта — это набор папок и правил размещения ассетов. Грамотная структура экономит время, упрощает поиск файлов и особенно важна при командной работе.

Основные принципы

  • Каждая система должна иметь свою папку.
  • Ассеты группируются по назначению, а не по типу.
  • Blueprints, Meshes, Materials и Textures всегда разделяются.
  • UI, VFX, Audio и Characters — отдельные категории.
  • Никаких ассетов в корне Content — всё только в папках.

Рекомендуемая структура проекта

Blueprints

Все игровые логические элементы.

  • Construction — сцены, уровни, вспомогательные BP для построения мира.
  • Game_Mechanics — игровые механики, системы, контроллеры, менеджеры.
  • UI — логика интерфейса (если не в отдельной папке).
  • Systems — инвентарь, квесты, сохранения, взаимодействие.

Environment

Всё, что относится к окружению.

  • Meshes — модели окружения.
  • Materials — материалы окружения.
  • Master — мастер-материалы.
  • Instance — инстансы материалов.
  • Textures — текстуры окружения.
  • Decals — следы, грязь, маркировка.

Characters

Персонажи и всё, что к ним относится.

  • Player — игрок, его анимации, BP, материалы.
  • NPC — враги, союзники, животные.
  • Animations — анимации, аним-графы, blend spaces.
  • Meshes — модели персонажей.
  • Materials — материалы персонажей.

VFX

Спецэффекты.

  • Envir_VFX — эффекты окружения (дым, огонь, дождь).
  • Weapon_VFX — эффекты оружия (вспышки, следы пуль).
  • Magic_VFX — магия, способности (если нужно).
  • Niagara — системы Niagara.

UI

Интерфейс.

  • Widgets — виджеты UMG.
  • Textures — иконки, элементы интерфейса.
  • Fonts — шрифты.

Audio

Звуки и музыка.

  • SFX — звуковые эффекты.
  • Music — музыка.
  • Mixes — миксы и настройки аудио.

Maps

Уровни проекта.

  • Main — основные игровые карты.
  • Test — тестовые уровни.
  • Sublevels — подуровни (Lighting, Geometry, Gameplay).

Data

Данные проекта.

  • DataTables — таблицы данных.
  • Curves — кривые.
  • Config — настройки систем.

FX / Materials / Meshes / Textures (общие)

Если ассеты используются в разных системах, их можно хранить в общих папках, но лучше избегать смешивания с Environment или Characters.

Важно

Приведённая структура — это лишь пример организации проекта. Каждый проект индивидуален и должен строиться под свои задачи, жанр, размер команды и особенности разработки. Используйте эту схему как отправную точку и адаптируйте её под свои нужды.
