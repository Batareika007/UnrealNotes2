Базовые понятия Unreal Engine

Этот раздел содержит базовые понятия, которые часто встречаются при работе в Unreal Engine. Они помогут лучше понимать структуру проекта, работу редактора и поведение объектов в сцене.

Preset (Пресет)

Набор заранее сохранённых настроек. В Unreal пресеты встречаются в материалах, пост‑процессинге, рендерах, освещении и других системах. Позволяют быстро применять готовые конфигурации.

World Outliner

Панель, в которой отображаются все объекты текущей сцены. Здесь можно искать, группировать, скрывать, блокировать и управлять любыми акторами.

Assets (Ассеты)

Любые ресурсы проекта: модели, текстуры, материалы, звуки, блюпринты, анимации и т. д.

Static Mesh (Статическая модель)

3D‑объект, созданный в стороннем редакторе (Blender, Maya и т. д.) и импортированный в Unreal. Имеет фиксированную геометрию, коллизию и может использоваться в сцене.

Collision (Коллизия)

Невидимые границы объекта, которые определяют, где игрок и другие объекты могут столкнуться.

  • Collision Player — простая коллизия, используемая для взаимодействия с игроком.
  • Collision Visibility — более точная коллизия, используемая для трассировки лучей (Line Trace).

Shaders (Шейдеры)

Набор инструкций, которые определяют, как отображаются материалы, освещение и эффекты.
«Компилировать шейдеры» — значит пересобрать визуальную часть материалов.

Engine Scalability Settings

Настройки качества графики. Определяют, как игра будет выглядеть на разных уровнях производительности (Low, Medium, High, Epic, Cinematic).

Режимы отображения (Viewport Modes)

Используются для анализа сцены и отладки.

  • Lit — финальный вид сцены с освещением и тенями.
  • Unlit — только геометрия и базовые цвета, без освещения. Удобно для поиска ошибок.
  • Wireframe — отображение сетки моделей. Помогает увидеть плотность геометрии.
  • Detail Lighting — показывает только освещение, без текстур. Полезно для анализа нормалей и теней.
  • Lighting Only — только свет, без материалов и рельефа.
  • Reflections — режим анализа отражений.
