Запекаем и создаём игру (Packaging & Cooking)

Перед сборкой проекта важно убедиться, что выбран правильный уровень, настроены параметры упаковки и отключено всё лишнее. Ниже приведён полный порядок подготовки игры к сборке.

1. Выбор главного уровня

Убедитесь, что выбран основной уровень, который будет использоваться как стартовый.

2. Настройки Packaging

Откройте Project Settings → Packaging.

3. Build Configuration

Выберите конфигурацию сборки:

  • Debug — используется, если проект содержит C++ код.
  • Development — подходит для проектов на Blueprint.
  • Test — промежуточный вариант: полноценная сборка, но с доступом к консоли отладки.
  • Shipping — финальная сборка для релиза. Максимальная оптимизация, отключена отладка.

Full Rebuild — обязательно включать для Shipping. Иначе движок возьмёт часть данных из предыдущей сборки.

4. Pak File

Use Pak File — все ассеты будут упакованы в один или несколько .pak файлов. Для защиты рекомендуется включить шифрование (Encryption).

5. Экономия места в сборке

Отключение лишних установщиков

Если игра распространяется через Steam или другие платформы, отключите:

Include prerequisites installer — снимите галочку.

Удаление ненужных плагинов

Отключите плагины, которые не используются, например:

  • OpenImageDenoise

6. Рекомендуемые настройки Packaging

  • Use Pak File
  • Share Material Shader Code
  • Shared Material Native Libraries
  • Create Compressed Cooked Packages
  • Exclude Editor Content When Cooking
  • Full Rebuild

7. Список карт (Maps)

В разделе Project → Packaging → List of Maps выберите только те уровни, которые нужны игре. Включите Cook Only Maps.

8. Исключение папок из сборки

В Project Settings → Packaging найдите параметр Directories to Never Cook и добавьте папки, которые не должны попадать в сборку.

9. Forward Shading

Включение Forward Shading может повысить производительность, но отключает Lumen. Используйте только если Lumen не нужен.

10. Отключение платформенных плагинов

Если игра не предназначена для мобильных или других платформ, отключите плагины:

  • Android
  • Apple
  • Linux
  • Chaos (если не используется физика Chaos)

Shipping Optimization

Для финальной сборки важно максимально оптимизировать игру, чтобы уменьшить размер, ускорить загрузку и повысить производительность. Основные рекомендации:

  • Включить Full Rebuild — пересобирает всё с нуля, исключая мусор от старых сборок.
  • Exclude Editor Content — исключает ассеты, используемые только в редакторе.
  • Create Compressed Cooked Packages — уменьшает размер файлов.
  • Use Pak File — упаковывает ассеты в единый архив, ускоряя загрузку.
  • Отключить ненужные плагины — снижает размер и ускоряет запуск.
  • Отключить платформы, которые не используются (Android, Apple, Linux).
  • Отключить Chaos, если физика Chaos не используется.
  • Включить Shader Sharing — уменьшает размер шейдеров.
  • Удалить неиспользуемые ассеты (ProjectCleaner).

Эти настройки помогают сделать финальную сборку компактной, быстрой и стабильной.

Cook, Cook by the Book и On the Fly — в чём разница

Unreal Engine использует процесс Cooking для подготовки ассетов к финальной сборке. Существует три основных режима:

Cook

Стандартный процесс подготовки ассетов. Используется при обычной упаковке проекта (Packaging). Подготавливает только нужные карты и ассеты.

Cook by the Book

Полная подготовка всех ассетов проекта по заранее заданному списку. Используется для стабильных, предсказуемых сборок. Это самый надёжный вариант для релиза.

Cook On the Fly (COTF)

Ассеты готовятся по мере необходимости, когда игра запрашивает их. Используется для разработки, тестирования и работы с большими проектами. Для релиза не рекомендуется.

Для финальной сборки всегда лучше использовать Cook by the Book.

Чек‑лист перед сборкой

  • Выбран правильный главный уровень (Game Default Map).
  • Указаны только нужные карты в Packaging → List of Maps.
  • Включён Full Rebuild.
  • Отключены лишние плагины и платформы.
  • Включён Use Pak File и Compression.
  • Отключён Include Prerequisites Installer (если Steam).
  • Папки, которые не нужны, добавлены в Directories to Never Cook.
  • Удалены неиспользуемые ассеты (ProjectCleaner).
  • Проверены ошибки в Output Log.
  • Сделан тестовый запуск Development сборки.

Если всё из списка выполнено — проект готов к финальной сборке Shipping.
